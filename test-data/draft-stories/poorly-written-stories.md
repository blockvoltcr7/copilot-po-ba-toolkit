# Draft Stories for Refinement

## Story 1: Portfolio Alert Dashboard

As a user, I want to see alerts on my dashboard so that I can do my job better.

### Acceptance Criteria
- Alerts should show up on the dashboard
- It should work properly
- Handle errors appropriately

### Size: 13

---

## Story 2: Risk Calculation Engine

As a financial advisor, I want the system to calculate portfolio risk.

### Acceptance Criteria
- Calculate risk scores for portfolios
- Update when market data changes

### Size: 8

---

## Story 3: Notification Service

As a financial advisor, I want to receive notifications when a client's portfolio drifts outside their risk tolerance, so that I can take corrective action before the next quarterly review.

### Acceptance Criteria
- Send email notification when risk score exceeds tolerance threshold
- Include client name, account number, current risk score, and tolerance level
- Allow advisor to snooze alerts for 7 days

### Size: 5

---

## Story 4: Compliance Reporting

As a compliance officer I want to see risk drift patterns across all advisors and clients so that we can identify systemic issues and demonstrate proactive monitoring to regulators.

### Acceptance Criteria
- Generate monthly compliance report
- Show all accounts that exceeded risk tolerance
- Include trend data
- etc.

### Size: 5

---

## Story 5: Mobile Push Notifications

As an advisor I want push notifications on my phone when critical risk alerts fire so I can respond quickly even when I'm not at my desk.

### Acceptance Criteria
- Push notification to iOS and Android
- Only for P1 and P2 severity alerts
- Advisor can configure quiet hours
- Should be secure
- Should be performant

### Size: 3
